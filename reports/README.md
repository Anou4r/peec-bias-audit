# Reports

Prompt-bias audit reports generated by `/peec-bias-audit`.

**Filename convention:**

- `prompt-bias-audit-YYYY-MM-DD.md` — the written audit
- `recommended-prompts-YYYY-MM-DD.csv` — companion CSV of Peec-ready prompt recommendations, one row per prompt, sorted by priority descending

Both files are written atomically by the same command run. If either already exists for today's date, a `-v2` / `-v3` suffix is used — no overwrites.

## Audit report structure

1. **Executive Summary** — overall corpus bias score, count of inflation-risk prompts, top 3 blind spots, recommended-prompt count (with sizing formula), plan headroom, honest-SoV estimate vs. reported.
2. **Methodology** — the 8 bias dimensions, the sizing formula, confidence caveats.
3. **Bias Dimension Heatmap** — one row per dimension with dominant category %, coverage entropy, corpus bias contribution, new prompts recommended.
4. **Inflation-Risk Prompts** — tracked prompts where the own brand wins but the framing is structurally self-flattering. Includes an `inflation_lift` estimate in percentage points.
5. **Per-Dimension Findings** — eight subsections, one per dimension, each with current distribution, representative prompts, blind-spot statement, and recommended additions (top 3).
6. **Simulated Honest Ranking** — directional projection of reported SoV vs. a blind-spot-balanced set.
7. **Execution Roadmap** — two parallel tracks. **Track A (Expand)** is a priority-ordered list split into "Add now" (fits current plan headroom) and "Unlocks at next tier". **Track B (Optimize in place)** pairs the lowest-marginal-value inflation-risk prompts with the top-N recommendations 1:1 for same-budget swaps. A decision matrix at the end tells the reader which mode fits their situation.
8. **Appendix** — full per-prompt classification table (all 8 dimension scores).

## Recommendation CSV schema

```
priority,targets_dimension,topic_suggestion,language,persona,buyer_stage,channel,industry,prompt_text,rationale,tier,swap_in_candidate,replaces_pattern
```

- `tier` ∈ `{swap_in, add_now, next_plan_tier}` — `swap_in` is the top-N paired 1:1 with retired inflation-risk prompts (no plan change); `add_now` is new additions that fit current plan headroom; `next_plan_tier` exceeds headroom under expand mode.
- `swap_in_candidate` ∈ `{yes, no}` — mirrors `tier=swap_in` for quick filtering.
- `replaces_pattern` — short descriptor of what the swap-in would retire (e.g. "home-language BOFU pricing-comparison redundant"). Empty for non-swap rows.

All text fields are double-quoted when they contain commas. Sorted by priority descending, with `swap_in_candidate=yes` first within priority ties.

## Committing your own reports

Generated reports are not committed in this repo by default — they are workspace-specific and may contain competitive information you don't want public. If you fork and want to keep internal runs alongside the code, use a private fork or extend `.gitignore` to exclude the dated artifacts:

```
reports/prompt-bias-audit-*.md
reports/recommended-prompts-*.csv
```
